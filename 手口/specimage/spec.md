# ソフトウェア要件定義書：日経225先物・オプション統合分析システム

## 1. プロジェクト概要

JPX（日本取引所グループ）が公表する「取引参加者別取引高（手口）」データをインポートし、証券会社ごとの売買動向を可視化するシステム。

特に、**「海外勢（米系・欧州系）vs 国内勢」の構図**や、**「オプション建玉を含めた実質的なデルタ金額（方向性）」**を算出し、相場の需給バランスを分析することを目的とする。

---

## 2. システム構成・技術スタック（推奨）

- **フロントエンド:** React.js または Vue.js（インタラクティブな表計算・ダッシュボード表示のため）
- **バックエンド:** Python (FastAPI または Flask) + Pandas（データ加工・科学計算用）
- **データソース:** ユーザー提供のCSV/Excelファイル（JPX公表データ）
- **外部データ連携:** 日経平均株価、TOPIX現在値、オプションデルタ値（API取得または固定値設定）

---

## 3. データ処理・ロジック要件

### 3.1. 入力データ仕様

ユーザーからアップロードされるファイル（CSV/Excel）のカラム定義。

- **Product Class:** 商品区分 (NK225F, NK225M, NK225E, TOPIXF 等)
- **Contract Issue:** 限月・権利行使価格情報 (例: NIKKEI 225 OOP P2602-52375)
- **Participant Name:** 証券会社名 (例: Goldman Sachs Japan)
- **Volume:** 取引数量
    - *必須処理:* **「売り(Sell)」ファイルと「買い(Buy)」ファイルを突き合わせ、ネット取引量（Net = Buy - Sell）を算出する処理を実装すること。**

### 3.2. 属性分類ロジック（証券会社マスタ）

Participant Name をキーとして、以下のカテゴリ（属性）を付与する。

| **カテゴリ** | **ラベル色** | **対象証券会社（代表例）** |
| --- | --- | --- |
| **米系証券 (US)** | 青系 | Goldman Sachs, Morgan Stanley, JP Morgan, BofA, Citigroup |
| **欧州系証券 (EU)** | 緑系 | ABN AMRO, Societe Generale, Barclays, BNP Paribas, UBS, Deutsche, HSBC |
| **日系証券 (JP)** | オレンジ系 | Nomura, Daiwa, Mizuho, SMBC Nikko, Mitsubishi UFJ MS |
| **ネット証券 (Net)** | 紫系 | SBI, Rakuten, Matsui, Monex, au Kabucom |

### 3.3. 商品情報のパース処理

Contract Issue 文字列を解析し、以下のメタデータを抽出する。

- **Call/Put:** C or P
- **限月:** 2602 → 2026年2月限
- **権利行使価格:** 52375 → 52,375円

## 4. 機能・画面仕様

### 4.1. 共通UI・ヘッダー

- **表示モード切替:** ドロップダウンリストで選択。
    - 「通常表示（フラット）」
    - 「分類表示（カテゴリ別）」 ※本仕様の重点項目
    - 「地域別集計（サマリー）」
- **CSV出力ボタン:** 現在表示している集計結果をダウンロード。

### 4.2. メイン画面：先物・デルタ一覧（分類表示モード）

先物手口とオプション手口を合算し、各業者が「強気（上目線）」か「弱気（下目線）」かを一覧表示する。

### A. テーブル構造・グルーピング

証券会社を属性ごとにブロック化して表示する。

- **グループヘッダー:** 「**米系証券 (X社)**」「**日系証券 (Y社)**」等の帯を表示。
- **ソート機能:** 各カラムヘッダー押下により、**グループ内での昇順・降順ソート**を行う。

### B. カラム定義と計算式

| **カラム名** | **データソース / 計算式** | **表示・スタイル** |
| --- | --- | --- |
| **証券会社** | Participant Name | 左寄せ |
| **日経225先物** | ラージ(NK225F)のNet枚数 | 整数。プラスは**青**、マイナスは**赤** |
| **日経225mini** | ミニ(NK225M)のNet枚数 | 同上 |
| **TOPIX先物** | TOPIX(TOPIXF)のNet枚数 | 同上 |
| **デルタ金額(億円)** | 下記「デルタ算出ロジック」参照 | **背景色：薄いオレンジ**<br>文字：**青(+)/赤(-)**<br>金額大(例:±100億超)は**太字** |

### C. デルタ金額 算出ロジック

各証券会社のポジションが、日経平均換算でいくら分の金額・方向性を持っているかを算出する。

```
Delta金額(円)=(ラージNet×P)+(ミニNet×P×0.1)+(TOPIX Net×Ptopix×1,000)+(∑OP Net×Delta×1,000)Delta金額(円)=(ラージNet×P)+(ミニNet×P×0.1)+(TOPIX Net×Ptopix×1,000)+(∑OP Net×Delta×1,000)
```

- 最終表示単位: **億円** (計算結果 ÷ 100,000,000)
- : 現在の日経平均先物価格（設定値 or API）
    
    ```
    PP
    ```
    
- : 現在のTOPIX先物価格
    
    ```
    PtopixPtopix
    ```
    
- : 各権利行使価格のデルタ値（※簡易実装の場合は、ATM=0.5とし、OTM/ITMで減衰させるモデルを使用するか、外部APIからGreeksを取得する）
    
    ```
    DeltaDelta
    ```

### 4.3. サブ画面：オプション手口マトリクス（ヒートマップ）

特定の属性、あるいは全社分のオプション詳細を表示する。

- **行:** 証券会社
- **列:** 権利行使価格ごとの Put Call Net
- **可視化:**
    - Net枚数がプラス（買い）の場合：**青色の背景（値が大きいほど濃い）**
    - Net枚数がマイナス（売り）の場合：**赤色の背景（値が大きいほど濃い）**
- **合計列:** 右端に「Put合計」「Call合計」「全Net合計」を表示。

## 5. 開発フローとデータマッピング詳細

### 実装ステップ

1. **Data Ingestion:** CSV読み込みとクリーニング、Buy/Sellの相殺計算処理の実装。
2. **Mapping:** 以下の証券会社名の正規化とカテゴリ付けの実装。
    - JPMorgan → 米系
    - Goldman Sachs → 米系
    - Morgan Stanley → 米系
    - BofA → 米系
    - Citigroup → 米系
    - ABN AMRO → 欧州系
    - Societe Generale → 欧州系
    - Barclays → 欧州系
    - BNP Paribas → 欧州系
    - UBS → 欧州系
    - Nomura → 日系
    - Daiwa → 日系
    - Mizuho → 日系
    - Mitsubishi UFJ → 日系
    - SBI / Rakuten / Monex / Matsui → ネット系
3. **Calculation:** デルタ金額計算エンジンの実装（価格データの入力フィールド作成を含む）。
4. **UI Construction:** 「分類表示」テーブルのレイアウト構築（CSS Grid/Flexbox使用）。

### エラーハンドリング・制約事項

- **データ欠損:** 一部の権利行使価格のみデータがある場合でも、マトリクスが崩れないようゼロ埋め処理を行うこと。
- **未分類の扱い:** マスタにない証券会社は「その他」カテゴリに分類し、システムが落ちないようにすること。
